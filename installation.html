You may use Random Walks Emulator as a library embedded into your own piece of software, or as a stand-alone application.

Note
Due to early stages of development, possibilities of a stand-alone application are limited to running a single example, however, they will definitely be broadened in the future.

At this moment, manual source code compilation is the only possible way to utilise this software as an application. As development goes further, the list of installation options will be enriched.

Option #1. Manual compilation

Prerequisites

  • Windows or any operating system from Unix family;
  • GCC 4.8.1 or newer (recommended GCC packages for Windows are MinGW and MinGW-w64);
  • Make 3.81 or newer (visit this or this for more info).

If you use Windows, both GCC and Make directories must be included into PATH variable.

Step-by-step manual

Step 1

Download the archive with the source code and unpack it.

Step 2

Open any shell in the directory where you have placed unpacked archive (PowerShell will be used for demonstration below). Make sure that Makefile file is available there. Run Make utility by calling make with no additional arguments in the shell.

Step 3

After Make utility halts, you shall be able to find a new folder in the directory where you have placed unpacked archive called Random Walks Emulator (build). Open it and find random_walks_emulator.exe. This is a console application, so you should run it via shell as well.
