Detailed Description

Emulator of RW-space.

RW-space is a collection of all possible continuous-time random walks over some graph represented by MetricGraph object.

Each RWSpace object at each moment of time is in one of the following states:

  • ready : RWSpace object is ready to start emulation;
  • active : RWSpace object is in process of emulation;
  • invalid : underlying MetricGraph object has been changed;
  • dead : underlying MetricGraph object has been deleted.

Run the emulation until the first \(\varepsilon\)-saturation moment

Runs the emulation of RW-space on an underlying MetricGraph object. Emulation starts at the specified vertex of the graph and ends when the first moment of \(\varepsilon\)-saturation occurs.

RW-space is said to be \(\varepsilon\)-saturated at time moment \(t_0\), if agents form an \(\varepsilon\)-net on each edge of the graph at \(t_0\).

No explicit checks of existence of the \(\varepsilon\)-saturation moments are made.

Parameters
start_vertexVertex where the initial agent instance will be spawned.
epsilonParameter \(\varepsilon\) of \(\varepsilon\)-saturation.
time_deltaTime step of emulation.
use_skip_forwardEnables a faster emulation algorithm; disable this option if you need precise positions of agent instances at each time step.
Returns
Time of the first \(\varepsilon\)-saturation moment; precision of the answer depends on the time step of emulation.
Note
The RWSpace object needs to be in the ready state in order to be run.
After the emulation halts, the RWSpace object is transferred into the invalid state.
Exceptions
logic_errorif the RWSpace object is either in active, or invalid, or dead states at the moment of function call.
invalid_argumentif initial vertex does not exist in the graph.
runtime_errorif CPU concurrency has been enabled, however, it is not supported by the platform.
