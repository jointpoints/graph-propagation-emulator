Quick reference

Getting started

The pipeline for Random Walks Emulator is quite simple and straightforward:

  1. Save your graph as a .gexf file. This format is supported by virtually all toolkits for graph processing, like Gephi or NetworkX.
  2. Write a scenario of your emulation.
  3. Run it and save the results.
Warning
Know that:
  • your graph must contain less than \(2^{32}\) vertices;
  • your vertex IDs must be non-negative integer numbers less than \(2^{32}\);
  • each edge must contain the weight attribute, it will be interpreted as a length of the edge;
  • isolated vertices will be ignored.

What is a .rweg file?

When you unpack the archive with the program and open the My graphs folder, you will see that it already contains two files:

  • Star2.rweg;
  • Star3.rweg.

These are sample star graphs saved in a special format .rweg (Random Walks Emulator Graph). Here is a brief comparison of .rweg with .gexf:

Criterion .gexf .rweg
File type XML-like plain text file Binary file
Perception by humans Human-readable Impossible to be read by humans
Memory consumption Files may have huge size Always considerably less than .gexf
Processing by Random Walks Emulator Fast Very fast

Basically, .rweg files only contain information necessary for emulation while .gexf files have various applications and, hence, may contain additional data related to visualisation, labelling, etc. All in all, the general rule is the following: if you want to always be able to look through the structure of your graph, keep it in the .gexf format; if you want to save space on your hard drive, keep it in the .rweg format.

Luckily, Random Walks Emulator is capable of converting files of these formats into each other. See Command list section below.

If you want to write your own program that works with .rweg format, consider the format specification.

What can this emulator do?

Currently Random Walks Emulator serves to only achieve one certain goal:

  • to compute the first moment of \(\varepsilon\)-saturation.

You may briefly read about \(\varepsilon\)-saturation here. More references are coming soon.

Command list

Here is the full list of currently implemented commands of Random Walks Emulator.

Command Brief
getconfig Prints current global parameters of emulator.
gexf2rweg Converts .gexf file into .rweg file.
help Prints the list of currently implemented commands.
run Runs the specified Random Walks Emulator Scenario file.
rweg2gexf Converts .rweg file into .gexf file.
setconfig Alters global parameters of emulator.
exit Exits the program.

Type <command> ? in Random Walks Emulator console to receive a full reference for a specific command.
