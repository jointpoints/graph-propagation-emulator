Detailed Description

A metric graph class.

Metric graph is defined as a graph \((V,E)\) where \(V\) is a set of vertices, \(E \subset V^2\) is a set of edges with each edge \(e \in E\) being associated with a certain interval \([0,l(e)]\), where \(l(e) > 0\) is called a length of the edge. Edge \((v,w) \in E\) is called directed ('from \(v\) to \(w\)'), if \((w,v) \notin E\). Such edges will be marked \(v \rightarrow w\) hereafter. Otherwise, if \((w,v) \in E\), the edge \((v,w) \in E\) is called undirected ('from \(v\) to \(w\)', or 'between \(v\) and \(w\)'). Such edges will be marked \(v-w\) hereafter. You may read more about metric graphs here.

This implementation also binds general theory by following additional restrictions:

  • \(V \subset \{0,1,2,...,2^{32}-1\}\) (i.e., vertices can only be represented as non-negative integers up to \(2^{32}-1\));
  • \(\{(v,w),(w,v)\} \subset E \Rightarrow l((v,w)) = l((w,v))\) (i.e., if one can go both ways, the distance between end points is the same).

\(v-w\) Existing edge \(v \rightarrow w\) is replaced with \(v-w\) of length \(l\).
\(v \leftarrow w\) \(v \rightarrow w\) Existing edge \(v \leftarrow w\) is replaced with \(v-w\) of length \(l\).
\(v-w\) Existing edge \(v \leftarrow w\) is replaced with \(v-w\) of length \(l\).

◆ fromGEXF()

void rwe::MetricGraph::fromGEXF ( std::string const  file_name)

Load graph from gexf file

This function loads graph from the specified gexf file and merges it with the existing one by adding all absent vertices, edges and updating their lengths and directions. If a file with the specified name does not exist, the original data is left unchanged.

The following additional requirements need to be met:

  • Version of the file must be 1.2draft;
  • Length of each edge must be stored in the weight attribute;
  • IDs of vertices must meet the requirements mentioned at the top of this page.

Isolated vertices are omitted. Mutual edges are treated as undirected. If edge with the same source and target is repeated multiple times, it will be treated as it is described in a table for updateEdge function in the respective order of repetitions.

Parameters
file_nameName of a source gexf file.
