Detailed Description

Emulator of random walks on metric graphs that calculates time of \(\varepsilon\)-saturation.

\(\varepsilon\)-saturation is defined as a state of wander when agent instances form an \(\varepsilon\)-net on each edge. Objects of this class are able to find out the moment of time (with certain precision) when \(\varepsilon\)-saturation of graph occurs for the first time.

Each Wander object at each moment of time is in one of the following states:

  • ready : Wander object is ready to start emulation;
  • active : Wander object is in process of emulation;
  • invalid : underlying MetricGraph object has been changed;
  • dead : underlying MetricGraph object has been deleted.

Concurrency type

Defines the mode of emulation:

  • none : consecutive (edges are updated one by one);
  • cpu : concurrent, using CPU (multiple edges are updated at the same time utilising multiple cores of CPU).
Warning
This is a future feature, currently it has no effect.

Run the emulation

Runs the emulation of random walks on an underlying MetricGraph object. Emulation starts at the specified vertex of the graph and ends when the first moment of \(\varepsilon\)-saturation occurs.

Parameters
start_vertexVertex where the initial agent instance will be spawned.
epsilonParameter \(\varepsilon\) of \(\varepsilon\)-saturation.
time_deltaTime step of emulation.
concurrency_typeEmulation mode.
Returns
Time of the first \(\varepsilon\)-saturation moment; precision of the answer depends on the time step of emulation.
Warning
concurrency_type is a future feature, currently it has no effect.
Note
The Wander object needs to be in the ready state in order to be run.
After the emulation halts, the Wander object is transferred into the invalid state.
Exceptions
logic_errorif the Wander object is either in active, or invalid, or dead states at the moment of function call.
invalid_argumentif initial vertex does not exist in the graph.
